\section{Baseline}

A single A100 with 40GB VRAM cannot fit fp32 at bs$=$8, and VICReg
collapses at bs$=$2 (Appendix Figure~\ref{fig:linear-mean}, grey). We
use bf16 to keep bs$=$8; this baseline (blue) converges to linear MSE
$0.243$ and is the starting point for every subsequent experiment.

\section{FFT preprocessing}

The default loader crops each frame to a square that fits the model's
stride; on the other Well datasets this discards information. We replaced
the crop with a band-limited FFT resize (Appendix
Figure~\ref{fig:fft-pipeline}) that preserves periodic boundary conditions
exactly. Even on \texttt{active\_matter}, whose native $256{\times}256$
grid needs no crop, FFT gave a consistent improvement — linear
$0.243 \to 0.212$ and k-NN $0.217 \to 0.174$ — so every subsequent
experiment uses FFT.

\section{Walking toward a spatiotemporal encoder}

With bf16$+$bs of 8$+$FFT fixed, we varied only the encoder. Each step
isolates one architectural choice.

\paragraph{SIGReg (negative control).}
Swapping VICReg for a LeWorldModel-style projection regulariser did not
converge: probe MSE \emph{rises} during pretraining (red dotted in
Appendix Figures~\ref{fig:linear-mean}--\ref{fig:knn-mean}). We revert to
VICReg.

\paragraph{Conv$+$Attn (one block).}
Adding a single self-attention block at the top of the original 3D-conv stack
edges linear to $0.251$ and k-NN to $0.259$ by epoch~17, barely moving
the FFT-only baseline.

\paragraph{Conv$+$Attn~$\times$6 (six blocks).}
Stacking six pre-norm transformer blocks at the same position does
improve the trajectory: linear descends
$0.753 \to 0.546 \to 0.370 \to 0.267$ across epochs $1, 3, 5, 7$ —
a steeper slope than the single-block run. HPC quota cut training at
epoch~7, before the curve could overtake the FFT-only baseline. So
depth helps, but only modestly. Both Conv$+$Attn variants share the
same defect: the attention sees only the post-conv \emph{spatial}
grid, since the strided 3D conv has already collapsed the temporal
axis. Depth alone cannot recover dynamics the tokeniser has thrown
away.

\paragraph{ViT3D.}
Replacing the conv stack with a single 3D patch embedding
($4{\times}16{\times}16$ patches) and six transformer blocks operating on
$4{\times}16{\times}16$ spatiotemporal tokens drops linear to
$\mathbf{0.107}$ and k-NN to $\mathbf{0.120}$ — roughly $3\times$ better
than every CNN variant we tried. The \emph{same} six transformer blocks
were inert on top of a conv stack and dominant on \emph{spatiotemporal}
patches: the clear win is in the tokenization.

\noindent
The five steps tell one story: VICReg needs bs$=$8 (so bf16); FFT helps
everywhere because it preserves periodic BCs; simply bolting attention onto a
strided conv stack cannot recover the temporal axis the
convs have already pooled away; ViT3D's spatiotemporal patches do.

\section{Discussion and future work}

Two ablations are needed to firm up the claim that tokenization, not
depth, drives the ViT3D win. First, Conv$+$Attn~$\times$6 was cut at
epoch~7 by HPC quota with the loss still descending, so its row mixes
architecture and undertraining; matching ViT3D's $\sim$$30$-epoch
budget would tell whether the deeper attention head plateaus or
overtakes the FFT-only baseline. Second, ViT3D itself uses six
blocks — the same depth as Conv$+$Attn~$\times$6 — so the tokenizer's
contribution should be isolated by running ViT3D with a single block,
and with no token mixing at all (patch embed + linear). If both still
beat every conv variant, the win is unambiguously in the
spatiotemporal patching; if the shallow variant collapses, depth and
tokenisation are co-dependent and the current takeaway weakens.
